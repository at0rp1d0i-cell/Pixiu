# Archive Reports

这里存放一次性实施总结、重构报告和阶段性交付说明。

这些文档的价值在于：

- 追溯某一轮改动当时是怎么被描述和验收的
- 理解为什么仓库里会出现某些兼容层或旧命名
- 核对某次集成时的背景，而不是指导当前开发

## 使用规则

- 先读 `docs/README.md`、`docs/overview/`、`docs/design/`
- 只有在需要追溯历史实现背景时，再回到这里
- 如果这里的某条信息仍然重要，应提升到活跃文档层，而不是继续让历史报告承担真相职责
