## 1. 启动时间约束

启动时间的瓶颈不是系统迭代速度，而是**市场日历**。OOS 窗口需要真实交易日推进，系统必须在等待期持续生成新假设，而不是空转。

---

## 2. 冷启动时间表

```
Day 1：爆发生成
  多轮迭代（目标：30+ 因子回测完成）
  深度反思模式启动
  → BP-1：反思库初始质量审查

Day 2-3：模拟盘启动
  Day 1 的 Top-10 候选进入模拟盘
  系统继续生成新假设

Day 4-7：并行推进
  模拟盘持续运行 + 系统持续迭代
  → BP-2：失败分布合理性审查

Day 8：中期评估
  → BP-3：OOS 退化趋势初判

Day 21：OOS 完成
  Day 1 因子有完整 21 天 OOS 数据
  → BP-4：Sharpe 退化率审查
```

---

## 3. 断点机制

| 断点 | 时机 | 审核内容 | 决策 |
|---|---|---|---|
| BP-1 | Day 1 结束 | 反思库初始质量 | Continue / Adjust Prompt |
| BP-2 | Day 5 | 失败分布合理性 | Continue / Reset Island |
| BP-3 | Day 8 | OOS 退化趋势 | Continue / 暂停 |
| BP-4 | Day 21 | Sharpe 退化率 | 进入纸测 / 重新调参 |

---

## 4. 实施备注

- 本规格属于运行节奏与运营层设计，不应与主架构总览混写。
- 实施前需要先完成控制平面持久化和基础 OOS 记录机制。
