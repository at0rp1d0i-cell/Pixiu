# Futures Guide

Purpose: Hold forward-looking design documents that are worth preserving but should not be mistaken for current runtime truth.
Status: active
Audience: both
Canonical: yes
Owner: core docs
Last Reviewed: 2026-03-18

`docs/futures/` 放的是“未来可能要做”的设计，不是“现在已经这样实现了”的设计。

适合放在这里的内容：

- 尚未进入主运行时的系统设计
- 中长期产品形态
- 前瞻验证层和商业化边界
- 需要保留思路、但不应进入默认阅读路径的设计稿

阅读顺序上，`docs/futures/` 永远晚于：

1. `docs/README.md`
2. `docs/overview/`
3. `docs/design/`
4. `docs/plans/` 中仍有效的执行文档
